# 🧭 План доработки MVP корпоративного мессенджера mesNation

> **Статус:** после реализации базовой связки «мобильный клиент + сервер + БД»  
> **Цель:** получить полноценный продукт для размещения в Google Play / App Store  
> **Принципы:** высокая архитектура, чистый код, расширяемость (уровень FAANG)

## 🗺️ Описание этапов

### Этап 1: Базовый функционал мессенджера ✅
- **Сервер:** регистрация, вход (JWT), профили пользователей, личные и групповые чаты, список участников, WebSocket для обмена сообщениями в реальном времени.
- **Клиент:** экраны авторизации, списка чатов, отправки сообщений, ответов на сообщения, редактирования и удаления сообщений, создание чатов с выбором участников.
- **База данных:** таблицы `users`, `roles`, `departments`, `tasks`, `messages`, `chats`, `chat_members`, `topics`. Все необходимые индексы и внешние ключи.

### Этап 2: Отправка файлов 🔧
- **Сервер:**  
  - Модернизировать `POST /api/upload` для приёма файлов из мобильного клиента.  
  - Генерировать миниатюры для изображений (библиотека `sharp`).  
  - Добавить эндпоинт для скачивания файлов с JWT-защитой.  
  - Новое поле `messages.thumb_url`.
- **Клиент:**  
  - Кнопка «Прикрепить файл» в `ChatScreen` (использовать `react-native-document-picker`).  
  - Отображение файлов в сообщениях: предпросмотр изображений, иконка для документов.  
  - Открытие файлов при нажатии через `Linking.openURL`.

### Этап 3: Супергруппы и топики 📅
- **Сервер:**  
  - Эндпоинты: `POST /api/chats/:id/topics`, `GET /api/chats/:id/topics`, `PATCH /api/topics/:id`.  
  - Логика WebSocket для переключения между топиками (сообщения привязаны к `topic_id`).  
  - Права: только создатель супергруппы может создавать/удалять топики.
- **Клиент:**  
  - Экран списка топиков внутри супергруппы.  
  - Возможность создавать, редактировать и удалять топики.  
  - Переключение между топиками в `ChatScreen` (фильтрация сообщений по `topic_id`).

### Этап 4: Улучшенный UI/UX 📅
- **Клиент:**  
  - Индикаторы «онлайн» и «печатает» (WebSocket).  
  - Загрузка аватаров пользователей (серверная часть + клиентский интерфейс).  
  - Тёмная тема (переключатель в настройках).  
  - Свайпы для ответа на сообщение (библиотеки `react-native-gesture-handler`, `react-native-reanimated`).

### Этап 5: Управление чатами и пользователями 📅
- **Сервер:**  
  - `PATCH /api/chats/:id` – изменение названия, is_supergroup.  
  - `DELETE /api/chats/:id` – покинуть чат или удалить группу (только создатель).  
  - `GET /api/users?search=...` – поиск пользователей по имени и email.  
  - Middleware для ролей (директор, менеджер, сотрудник) при создании групп.
- **Клиент:**  
  - Экран «Информация о чате»: смена названия, добавление/удаление участников, выход из чата.  
  - Улучшенный поиск пользователей при создании чата.  
  - Экран профиля пользователя (смена имени, аватара).

### Этап 6: On‑premise развёртывание и публикация 📅
- **Сервер:**  
  - Docker Compose для быстрого развёртывания.  
  - Инструкция `INSTALL.md` для системных администраторов.  
  - Переменные окружения для настройки хоста, порта, секретов.
- **Клиент:**  
  - Экран «Подключение к серверу» при первом запуске (сохранение адреса в AsyncStorage).  
  - Сборка и подпись APK/AAB для Android, настройка сертификатов для iOS.  
  - Публикация в Google Play и App Store.

### Этап 7: CRM‑часть и AI‑чат (после релиза) 📅
- **Туду‑лист с иерархией «ЧУМ»:** создание задач, дедлайны, права по ролям и отделам.
- **Календарь и KPI:** визуализация задач, прогресс‑бары выполнения планов.
- **Клиентская база и синхронизация с 1С.**
- **AI‑чат (RAG):** чат‑бот, отвечающий на вопросы сотрудников по загруженным документам (архитектура описана в `хотелки.md`).
