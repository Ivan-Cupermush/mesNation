# 🧠 CRM & Корпоративный мессенджер — структура проекта

> **Проект:** `mesNation` — CRM для малого и среднего бизнеса с иерархическим задачником, мессенджером и интеграцией с 1С.  
> **Репозиторий:** монорепозиторий, всё в одной ветке `main`.

---

## 🌟 Что мы строим

- **Туду-лист с иерархией «ЧУМ»** – задачи создаются по уровням подчинения, отделы не пересекаются.
- **Календарь** – визуализация задач по дедлайнам.
- **Мессенджер** – личные и групповые чаты, закреплённые сообщения.
- **Клиентская база** – синхронизация с 1С.
- **Результаты и KPI** – прогресс-бары продаж, статистика выполнения.
- **On‑premise** – сервер ставится на оборудовании заказчика.

## 🖥️ `server/` – Бэкенд

Сейчас это работающий каркас Express‑сервера.  
Запуск: `npm run dev` → слушает порт `5000`, возвращает `{"message":"CRM API работает!"}`.

### Файлы и папки

| Путь | Назначение |
|------|-----------|
| `server/package.json` | Зависимости (`express`, `cors`, `dotenv`) и скрипт `dev` через `ts-node-dev` |
| `server/package-lock.json`| Фиксация версий |
| `server/tsconfig.json` | Компиляция в ES2020, commonjs, включён `strict` |
| `server/src/index.ts` | Точка входа: инициализация Express, middleware, тестовый маршрут |

**Ближайшее будущее:** модели БД, роуты, WebSocket для чата, интеграция с 1С.

## 📚 `docs/` – Документация

Центральное хранилище идей, планов и рабочих записей.

| Файл / папка | Назначение |
|--------------|-----------|
| `docs/README.md` | Пояснение содержимого |
| `docs/хотелки.md` | **Основной документ:** подробное описание функций, архитектурные диаграммы (Mermaid), технологический стек, план на 3 месяца (6 спринтов) |
| `docs/Итоги работы` | Журнал за дни разработки (установка окружения, создание первого бэкенда) |
| `docs/requirements/` | Заготовка под будущие требования |
| `docs/architecture/` | Заготовка под архитектурные схемы |
| `docs/design/` | Заготовка под дизайн‑макеты |
| `docs/development/` | Заготовка под гайды для разработчиков |
| `docs/testing/` | Заготовка под тестовую документацию |
| `docs/meetings/` | Заготовка под протоколы встреч |
| `docs/reports/` | Заготовка под отчёты о прогрессе |

> 💡 **Почему так:** папки‑заготовки созданы, чтобы сразу придать проекту взрослую структуру и упростить навигацию новым участникам.

## 🔄 Как развивается проект

1.  **Legacy‑этап** – первая версия бэкенда (`mesNation-backend`) описана в `docs/Итоги работы`; код заменён на текущий.
2.  **Монорепозиторий** – всё объединено в ветке `main`.
3.  **Будущие шаги:**
    *   Наполнение `server/` бизнес‑логикой, подключение PostgreSQL.
    *   Инициализация `mobile/` приложения.
    *   Интеграция с 1С, внедрение AI‑чата (RAG) согласно плану.

---

## 💡 Правила и соглашения

- **Единая ветка `main`** – вся работа идёт здесь.
- **Markdown** для всей документации (совместимо с Obsidian).
- **TypeScript** на клиенте и сервере.
- **Без `.env` в репозитории** – секреты только локально.
- **Папка `docs/`** – живая база знаний, пополняется по ходу проекта.
