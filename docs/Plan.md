# 🧭 План доработки MVP корпоративного мессенджера mesNation

> Статус: после реализации базовой связки «мобильный клиент + сервер + БД»  
> Цель: получить полноценный продукт для размещения в Google Play / App Store  
> Принципы: высокая архитектура, чистый код, расширяемость (уровень FAANG)

---

## 🎯 Целевой функционал MVP

1. **Регистрация и аутентификация**  
   - Ввод конфигурации сервера (адрес компании) при первом запуске.  
   - Создание профиля: уникальный юзернейм, имя, аватар (автогенерация по буквам).  
   - JWT‑токены, защищённые эндпоинты.

2. **Чаты (основной экран)**  
   - Список чатов с последним сообщением.  
   - Личные и групповые чаты.  
   - Ответ на сообщение (свайп / долгое нажатие → цитирование).  
   - Удаление сообщения (только у себя / у всех).  
   - Закрепление сообщений (только для администраторов).  
   - Отправка файлов (изображения, документы) с предпросмотром и скачиванием.  
   - Индикаторы «онлайн», «печатает».

3. **Серверная часть**  
   - On‑premise установка на сервере компании.  
   - REST API + WebSocket (Socket.IO).  
   - Хранение всех данных в PostgreSQL.  
   - Загрузка файлов с сохранением на диск / в объектное хранилище.  
   - Конфигурационный файл для подключения клиентов.

4. **Развёртывание**  
   - Инструкция по установке сервера (Docker Compose).  
   - CI/CD для автоматической сборки мобильных приложений.  
   - Публикация в Google Play и App Store.

---

## 🏗️ Архитектурные стандарты

- **Монорепозиторий** с чётким разделением: `server/`, `mobile/`, `docs/`.  
- **TypeScript** на всех слоях.  
- **Сервер**: трёхслойная архитектура (контроллеры → сервисы → репозиторий).  
- **Клиент (React Native)**: функциональные компоненты, хуки, разделение на экраны / компоненты / сервисы.  
- **База данных**: нормализованные таблицы, индексы, миграции.  
- **Безопасность**: пароли хешируются (bcrypt), JWT с refresh‑токенами, валидация входных данных.  
- **Тестирование**: юнит‑тесты для сервисов и критической логики.

## 📋 Пошаговый план

### Этап 1. Аутентификация и профиль
- **Сервер**:  
  - Модель `User` (username, password_hash, display_name, avatar_url, created_at).  
  - Эндпоинты: `POST /auth/register`, `POST /auth/login`, `GET /auth/me`.  
  - Middleware проверки JWT.  
- **Клиент**:  
  - Экран входа / регистрации.  
  - Экран профиля (смена аватара, имени).  
  - Защищённый роутинг.

### Этап 2. Список чатов и создание групп
- **Сервер**:  
  - Модели `Chat` (id, type, name, created_by) и `ChatMember`.  
  - Эндпоинты: `GET /chats`, `POST /chats` (создание личного или группового), `POST /chats/:id/members`.  
  - При создании группового чата автоматически добавлять создателя.  
- **Клиент**:  
  - Экран со списком чатов, отсортированным по последнему сообщению.  
  - Кнопка «Создать чат» с выбором участников.

### Этап 3. Улучшенный чат (ответы, удаление, закрепление)
- **Сервер**:  
  - В таблице `messages` добавить поля: `reply_to`, `deleted_for`, `pinned`.  
  - Эндпоинт `DELETE /messages/:id` (помечать удалённым для всех или только для себя).  
  - Эндпоинт `PATCH /messages/:id/pin`.  
  - При отправке сообщения с `reply_to` сервер валидирует, что исходное сообщение существует.  
- **Клиент**:  
  - UI для ответа: при свайпе или долгом нажатии появляется поле ввода с цитатой.  
  - Кнопка «Удалить» с выбором «У всех» / «У меня».  
  - Закреплённые сообщения сверху (только для админов).  
  - Обновление интерфейса через WebSocket при удалении/закреплении.

### Этап 4. Отправка файлов
- **Сервер**:  
  - Настроить `multer` для сохранения в `uploads/` (или в S3‑совместимое хранилище для продакшена).  
  - Генерировать уникальное имя файла, сохранять в БД `file_url`, `file_name`, `file_size`.  
  - При запросе истории отдавать временные ссылки (если файлы защищены).  
- **Клиент**:  
  - Кнопка «Прикрепить файл» → выбор из галереи / файлов.  
  - Отображение изображений с предпросмотром, документов с иконкой.  
  - Скачивание файлов по нажатию.

### Этап 5. Конфигурация сервера и деплой
- **Сервер**:  
  - Создать `docker-compose.yml` с PostgreSQL и Node.js.  
  - Инструкция `INSTALL.md` для заказчика.  
  - Переменные окружения для настройки хоста, порта, секретов.  
- **Клиент**:  
  - При первом запуске экран «Подключение к серверу» с полем ввода адреса (сохраняется в AsyncStorage).  
  - Возможность сменить сервер в настройках.

### Этап 6. Публикация в магазины
- **Android**: сборка APK/AAB с подписью, загрузка в Google Play Console.  
- **iOS**: настройка сертификатов, сборка через Xcode, загрузка в App Store Connect.
